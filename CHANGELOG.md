# Changelog

## [2.2.3] - 2026-07-21

### Fixed

- 章节规划缺号时不再把所有占位章节追加到最后一卷；缺号会按相邻分卷起始章节归位并按编号排序。
- 自动合并 Agent 重复输出的同编号分卷；旧项目可在作者确认后安全修复重复卷、章节乱序、位置异常和缺号。
- 章节结构修复保留已有章节 ID、正文、场景和版本记录，并在操作前创建永久特殊快照。
- 总编识别“推进工作流”等明确指令后生成待确认操作卡，不再仅用聊天文字声称已推进；作者点击执行后才真正创建生成任务。

### Verification

- 新增章节规划归一化、旧章节树修复和总编确认执行回归；真实 80 章项目数据库副本验证 80 个章节 ID 全部保留，6 卷和第 1 至 80 章连续归位。
- 后端 183 项、前端 42 项、Playwright 5 项测试通过；Ruff、strict mypy、TypeScript 与生产构建通过，最终 ZIP 解压后的控制台和 GUI 生命周期自检通过。

## [2.2.2] - 2026-07-21

- 项目删除失败时在首页显示真实错误并保留项目，不再静默无响应。
- 修正侧栏折叠后按钮提示仍显示“收起侧栏”的状态错误。

### Verification

- 新增当前桌面版 104 个可达按钮的 AST 契约检查，禁止无点击处理器或无表单提交处理器的按钮进入构建。
- 新增项目删除数据库回归与 Playwright 持久化回归，并扩充主页、侧栏、模型中心和自定义 API 交互测试。
- 后端 180 项、前端 41 项测试通过；Ruff、strict mypy、TypeScript 检查通过。

## [2.2.1] - 2026-07-21

### Added

- 工作室生成、AI 对话和文风提取按真实模型上下文窗口预留输出空间并自动计算输入 Token 预算。
- 半成品长篇分析使用章节分层索引与最多 32 片的共享 Map-Reduce 汇总，避免一次提交整本原文。
- ContextBuilder 检索结果、压缩数量和分片数量写入生成元数据与任务模型理由，便于核对本次实际上下文。

### Fixed

- Provider 返回 `context_too_long` 时会逐级压缩低优先级中段并自动重试，同时保留任务开头和作者结尾要求。
- 修复右侧 AI 对话长回复撑高页面、遮挡主工作区和输入框的问题；消息区现在独立滚动，输入框固定在右栏底部。
- 修复 1024px 响应式布局下绝对定位右栏同时继承 `height: 100%` 导致底部超出视口的问题。

### Verification

- 后端 179 项、前端 32 项测试通过；4 个 Playwright E2E 覆盖小窗口上下文、压缩重试、半成品续写和 18 轮长对话滚动布局。

## [2.2.0] - 2026-07-21

### Added

- 新增半成品小说续写项目，可从 TXT、Markdown、Word、PDF、粘贴正文或已有项目导入。
- 导入时识别卷章正文，创建可编辑工作副本，并永久保存不可编辑的完整原文工件与特殊快照。
- 新增“导入与解析 → 资料审核 → 大纲补建 → 续写规划 → 正文续写 → 全文审阅 → 完成”七阶段流程。
- 多 Agent 提取章节结构、世界观、人物关系、时间线、伏笔、原文文风和未完剧情线，并反向补建分卷、章节和场景大纲。
- 支持作者方向与 AI 提案切换、总字数/章节/卷数设置、续写当前章或下一章选择，以及规划批准后建立未来卷章。

### Safety

- 所有分析、大纲和续写规划必须逐项审核；重大连续性冲突会暂停后续生成，直到作者明确处理。
- AI 修改已有正文只生成待审核提案；接续当前章采用追加写入，写入前自动创建保护快照。

### Verification

- 新增导入、PDF 文本提取、只读原文、审核门禁、未来章节建立、追加写入、冲突暂停和前端导入向导回归测试。

## [2.1.3] - 2026-07-21

### Added

- 全文审阅阶段继续显示逐章正文编辑器，支持自动保存、手动保存、重写、局部修改及章节增删。
- 增加需要作者确认的旧项目章节结构修复：操作前创建永久特殊快照，错误占位章软删除并补齐缺失章节。

### Fixed

- 章节规划不再把“章节规划师”“场景规划师”等 Agent 标题识别为正文章节。
- 长篇章节与场景规划按每 10 章分批调用并合并，避免 80 章项目因单次输出上限只得到前 7 章。
- 章节树严格补齐项目设定的章节数，并使用场景规划师成果建立场景；缺失内容使用可编辑占位结构兜底。
- 生产数据库关闭自动刷新时，正文批准后的阶段推进、场景合并和特殊快照现在显式刷新后再查询。

### Verification

- 真实 v2.1.2 数据库在线备份副本验证：识别 80 章目标、保留已有第 1 至 7 章、移除错误占位并补齐第 8 至 80 章，原正文永久快照保留。
- 后端、前端、类型检查、生产构建与 Playwright 审阅阶段编辑回归通过。

## [2.1.0] - 2026-07-20

### Added

- 双入口创作流程，以及定位主题、世界规则、文风边界、人物关系、剧情、分卷、章节和场景的独立 Agent 审核项。

## [1.0.0] - 2026-07-18

### Added

- 小说项目、写作工作台、资料库、版本、回收站和中文示例数据。
- 五种模型协议、可编辑 Provider 预设、自定义 JSON/SSE/NDJSON/Chunked/Raw Adapter。
- 能力探测与降级、Route、限流、预算、熔断、成本和调用账本。
- 可版本化 Agent、React Flow DAG、并行调度、SSE、取消、重试和派生运行。
- 可解释上下文检索、FTS、Pin、分类、Token 预算和 Provider 数据边界。
- 人工审批、Diff、状态提取、逐项变更、冲突解决和事务写回。
- 完整备份/恢复、九种导出、诊断包、日志管理和恢复状态页。
- Windows 安装包、卸载器、免安装 ZIP、PyInstaller 运行时和 SHA-256 清单。

### Security

- 本地 Host/Origin 限制、安全 Header、SSRF 与 DNS rebinding 防护、凭据隔离、Secret 脱敏、受限模板和备份安全校验。
- 恢复与写回失败均使用事务回滚，生产桌面包关闭开发 API 文档。

### Performance

- Workflow 历史游标分页、流式 rAF 批处理、React Flow 可见元素渲染和实体状态批量查询。
- 加入 100 节点 DAG、10 万字符章节和 1,000 实体检索回归。

### Fixed

- 桌面启动器不再把 Edge 的首个转交进程退出误判为应用关闭；后台现在跟随真实 Novel Agent Studio 窗口生命周期。
- 安装器会等待卸载器的临时清理进程真正结束，避免卸载后立即重装时快捷方式或卸载注册表项发生竞态。
