# Performance Audit

审计日期：2026-07-21
版本：2.2.2

## 验收样本

以下数字来自 Windows 11、Python 3.12、SQLite 本地测试进程，记录 pytest call 时间并排除临时数据库 fixture 启动。不同硬件、杀毒软件、磁盘和实际模型延迟会改变结果。

| 场景 | 验收约束 | 本次 call 时间 | 结果 |
| --- | --- | --- | --- |
| 100 节点 DAG 编译，另含 125 条历史三页游标读取 | 编译小于 2 秒；分页 50/50/25 且不重复 | 约 0.04 秒 | 通过 |
| 100,000 个中文字符章节自动保存与字数统计 | 小于 5 秒 | 约 0.01 秒 | 通过 |
| 1,000 实体及最新状态检索 | 小于 5 秒，SQL 语句不超过 5 条 | 约 0.10 秒，语句不超过 5 条 | 通过 |

性能专项共 3 项，`3 passed in 4.17s`；其中约 3.24 秒是三个独立临时 SQLite fixture 的建库时间。

## 已实施优化

- Workflow 历史使用 `before_id` 游标分页，前端按需无限加载，避免一次拉取全部运行记录。
- 流式事件在浏览器中按 `requestAnimationFrame` 批量提交，减少高频 token 导致的 React 重渲染。
- React Flow 启用 `onlyRenderVisibleElements`，100 节点画布只渲染视口附近元素。
- 实体最新状态使用窗口查询批量读取，消除 1,000 实体场景的 N+1 查询。
- 章节编辑、上下文和运行事件均有明确结果数、输出大小或历史页大小边界。
- Vite 路由级拆包；最终生产构建无 chunk-size 警告。
- 长篇半成品资料先按章节建立有限分层索引，再以共享分片提取供多个 Agent 汇总，避免七个 Agent 各自重复读取整本原文。
- 工作室 Prompt 在网络调用前按模型窗口压缩，阻止确定会失败的超限请求进入 Provider 重试队列。

## 非承诺范围

- 实际模型调用通常由网络和 Provider 延迟主导，本地结果不代表云模型吞吐量。
- SQLite 和单进程调度面向个人创作，不是分布式并发服务。
- 10 万字符和 1,000 实体是发布回归样本，不是无限规模保证。
- FTS5 是词法检索；更大的中文语料可能需要外部分词或向量检索扩展。
