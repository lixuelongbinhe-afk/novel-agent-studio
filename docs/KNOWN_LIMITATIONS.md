# Known Limitations

版本：2.1.2

- 正式桌面包面向 Windows 10/11 x64，并依赖系统可用的 Microsoft Edge 或 WebView2 Runtime。源码 Web 版可在其他平台开发运行，但未发布对应安装包。
- 安装包尚未使用商业代码签名，Windows SmartScreen 可能显示“未知发布者”。请用 `SHA256SUMS.txt` 校验来自本项目的产物。
- 应用是本机单用户工具，没有登录、TLS、角色权限、租户隔离或公网部署加固。不要把本地端口转发到其他设备或互联网。
- SQLite 数据库、日志和完整备份默认未加密。完整备份包含未发布正文；卸载保留的数据也不会自动安全擦除。
- API Key 值不进入数据库，可保存在 Windows Credential Manager 或由环境变量提供；同一 Windows 账户下有足够权限的软件仍位于本地信任边界内。
- 关闭窗口选择“转入托盘继续”时任务继续；选择“停止并退出”会终止进程。下次启动会把遗留的 Workflow 和生成任务明确标记为 interrupted/failed，不会假装离线完成。
- 调度、实时取消、并发/RPM/TPM 预留和事件扇出只在单个本地进程内生效，不是分布式配额系统。
- SQLite FTS5 是词法检索，中文分词和语义召回有限；Embedding Retriever 只有扩展接口，默认没有外部向量数据库。
- Token 和成本可能来自 Provider usage、tokenizer 或本地估算。未知价格保持未知；用户需维护当前模型价格、上下文窗口和 Provider 条款。
- 自动化测试仅调用 Mock/Fake Provider。DeepSeek、Grok、Ollama 和其他真实服务的可用性、账户权限、地区限制、内容政策和响应格式可能变化。
- 结构化输出可降级为本地验证和一次有限修复；无法满足 Schema 时进入人工处理，不会无限重试。
- 已有部分流式正文后不会静默切换模型并拼接，因而某些上游中断会直接失败并保留部分输出。
- 备份恢复采用全量替换策略，并要求预览和 SHA-256 确认；不是多设备增量同步或协同合并工具。
- 自动与倒计时续写只在桌面程序仍运行（包括托盘状态）时继续，不是关机后运行的云任务。
- 没有自动更新器。升级需运行新安装包覆盖安装，或用新版便携目录替换程序文件并保留/迁移 `data`。
- 未进行独立第三方渗透测试、无障碍认证或超出发布样本规模的长期压力测试。
